# Doc API
A `doc` represents an instance of `unified-doc` with methods to manage `content`.

This section provides a quick overview of a `doc`.  More details are provided in later sections with interactive code playgrounds to test and preview live code changes.

## Initialize
Simply provide the `content` and `filename` options to initialize a `doc` instance.

```js
const doc = unifiedDoc({
  content: '> **some** markdown content',
  filename: 'doc.md',
});
```

## Options

A `doc` is configurable with the following options:

```ts
interface Options {
  content: string;
  filename: string;
  annotations?: Annotation[];
  annotationCallbacks?: AnnotationCallbacks;
  compiler?: Compiler;
  plugins?: Plugin | Plugin[];
  sanitizeSchema?: SanitizeSchema;
  searchAlgorithm?: SearchAlgorithm;
  searchOptions?: SearchOptions;
}
```

## Methods
A `doc` provides the following useful methods to manage `content`:

### `doc.compile(): VFile`
Returns the compiled contents of a `doc` that is used by renderers to render the `doc`.

### `doc.file(extension?: string): FileData`
Returns file data for supported file extensions, allowing convenient conversions of content formats.

### `doc.parse(): Hast`
Returns the [hast][hast] representation of the source content.

### `doc.search(query: string, options?: object): SearchResultSnippet[]`
Provides a unified interface to easily search the `textContent` of a `doc` and return search result snippets.

### `doc.textContent(): string`
Returns the `textContent` of the source content.
